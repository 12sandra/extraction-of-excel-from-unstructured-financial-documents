## 🚀 Model Evolution

### 🔹 Version 1: Heavy Model

* OCR: PaddleOCR + GOT
* High accuracy
* Slow and resource-intensive

---

### 🔹 Version 2: Light Model

* OCR: DocTR + PaddleOCR
* Multi-OCR support
* Faster but architecturally inconsistent

---

### 🔹 Version 3: GST4

* Modular pipeline design
* Components:

  * OCR Engine
  * LLM Engine
  * Schema Mapper
  * Validator
* Introduced structured processing

---

### 🔹 Version 4: GST5

* Improved extraction accuracy
* Better handling of noisy OCR outputs
* Enhanced table reconstruction
* More stable pipeline

---

### 🔹 Version 5: GST2A (Final Model)

* Achieved correct GST data extraction ✅
* End-to-end working pipeline
* Structured and validated output
* Ready for real-world usage
